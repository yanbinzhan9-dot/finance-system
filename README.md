# 财务系统

本地版亚马逊合伙财务系统，适合记录詹、蓝之间的项目投入、垫付款、凭证和周期结算。

## 打开方式

双击 `start-finance-system.bat`，浏览器会打开：

```text
http://127.0.0.1:8765/index.html
```

也可以直接打开 `index.html` 使用。

## 已实现

- 看板：累计投入、首期 50 万进度、詹/蓝实际投入、首期应承担和当前差额。
- 记账：支出、收入、退款、调整、结算支付。
- 投入规则：首期 50 万詹 60%、蓝 40%；超过部分规则待双方确认。
- 人员规则：收入、支出和结算统一用经手/支付/收款人记录。
- 币种：默认人民币，支持美元，系统会按账目日期自动获取 USD/CNY，失败时可先保存后续补齐。
- 凭证：每条账目支持多张图片或 PDF 文件，点击可打开预览。
- 搜索：按关键词、类型、人员、币种、日期筛选。
- 结算：记录双方实际转账，自动更新双方实际投入和当前差额。
- 备份：支持导出和导入 JSON 数据。

## 数据说明

数据保存在项目目录的 `data/finance-data.json`，凭证保存在 `data/uploads/`。Chrome 和 Codex 右侧浏览器打开同一个本地地址时，会读取同一份数据。

如果旧版本已经在某个浏览器里录入过数据，刷新新版页面时会自动尝试把该浏览器里的旧数据迁移到本地数据文件。

## 线上部署

项目已准备好 Render 部署文件：

- `render.yaml`
- `Procfile`
- `requirements.txt`

推荐方式：

1. 把本目录作为一个 GitHub 仓库上传。
2. 在 Render 里新建 Blueprint，选择这个仓库。
3. 设置环境变量 `FINANCE_PASSWORD` 为你们两人使用的访问密码。
4. Render 会分配一个 `onrender.com` 地址，不需要你自己买域名。

线上版本会使用 `render.yaml` 里的 1GB 持久磁盘保存数据和凭证。部署后，双方打开同一个 Render 地址即可共同录入和修改。

汇率使用 Frankfurter 公共历史汇率接口，美元账目会按账目日期自动获取 USD/CNY，并写入备注。
